### **Adding a Logical Volume (LV) in LVM**  

Logical Volumes (LV) in LVM allow for flexible disk space management by creating virtual partitions inside a Volume Group (VG).  

---

### **Check Existing Volume Groups**  
Before creating a Logical Volume, check the available Volume Groups (VG):  
```bash
sudo vgs
```
If you haven't created a Volume Group yet, you can create one using a Physical Volume (PV):  
```bash
sudo vgcreate vg01 /dev/sdb
```
Verify the VG:  
```bash
sudo vgdisplay vg01
```

---

### **Create a New Logical Volume**  

**Create an LV with a specific size (e.g., 10GB):**  
```bash
sudo lvcreate -L 10G -n lv_data vg01
```

**Create an LV using all available space:**  
```bash
sudo lvcreate -l 100%FREE -n lv_data vg01
```
Verify the created LV:  
```bash
sudo lvs
sudo lvdisplay /dev/vg01/lv_data
```

---

### **Format the New Logical Volume**  

Before using the Logical Volume, it must be formatted:  

**Format as ext4:**  
```bash
sudo mkfs.ext4 /dev/vg01/lv_data
```

**Format as XFS (recommended for RHEL-based systems):**  
```bash
sudo mkfs.xfs /dev/vg01/lv_data
```

---

### **Mount the Logical Volume**  

Create a mount point:  
```bash
sudo mkdir /mnt/data
```
Mount the Logical Volume:  
```bash
sudo mount /dev/vg01/lv_data /mnt/data
```
Check if it's mounted correctly:  
```bash
df -h
```

---

### **Make the Mount Persistent**  

To ensure the Logical Volume mounts automatically after a reboot, get its UUID:  
```bash
blkid /dev/vg01/lv_data
```
Edit the `/etc/fstab` file:  
```bash
sudo nano /etc/fstab
```
Add the following line (replace `UUID=xxxx` with the actual UUID):  
```ini
UUID=xxxx /mnt/data ext4 defaults 0 2
```
Apply the changes:  
```bash
sudo mount -a
```

---

### **Expand an Existing Logical Volume**  

If you need to increase the size of an existing Logical Volume:  
```bash
sudo lvextend -L +5G /dev/vg01/lv_data
```
Or use all available space:  
```bash
sudo lvextend -l +100%FREE /dev/vg01/lv_data
```

After extending, resize the filesystem:  

**For ext4:**  
```bash
sudo resize2fs /dev/vg01/lv_data
```

**For XFS:**  
```bash
sudo xfs_growfs /dev/vg01/lv_data
```

---

### **Verify the Changes**  

Check the updated disk space:  
```bash
df -h
sudo lvs
```
